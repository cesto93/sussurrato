# Sussurrato — Agent guide

Android app for on-device audio transcription. Supports two backends:
- **LiteRT-LM** (`.litertlm` models) — Google's on-device ML engine
- **llama.cpp + Qwen3-ASR** (`.gguf` models) — via the `:llama` native module

## Quick reference

```bash
./gradlew :app:assembleDebug              # build full APK
./gradlew :llama:assembleDebug            # build only the llama library AAR
./gradlew test                            # run unit tests
./gradlew :app:connectedDebugAndroidTest  # run instrumented tests (device/emulator)
```

### Key architectural details

- **`TranscriptionViewModel`** manages dual-engine lifecycle: routes `.gguf` files to `AsrEngine` (llama.cpp) and `.litertlm` files to LiteRT-LM `Engine`.
- **Model download flow**: Android `DownloadManager` enqueues the decoder GGUF file; on success, downloads the mmproj companion file; then calls `initEngine` with both paths.
- **Audio pipeline**: `MediaExtractor` + `MediaCodec` decodes any audio to 16 kHz mono PCM F32. For LiteRT-LM this is wrapped as WAV; for llama.cpp it's passed as `FloatArray` directly to JNI.
- **llama.cpp module (`:llama`)**: builds `libsussurrato-llama.so` via CMake with arm64-v8a + x86_64. Uses mtmd multimodal API (not text-only ai_chat). Compiled with `GGML_CPU_ALL_VARIANTS=ON` (no OpenCL, no BLAS, no Metal).

## Build system

- **Gradle 9.4.1** via wrapper, **AGP 9.2.1**, **Kotlin 2.2.10** with Compose compiler plugin.
- **Min SDK 28, target/compile SDK 36.** Java 17 source/target compatibility (bumped from 11 for the `:llama` module).
- **Jetpack Compose + Material3** using BOM `2026.02.01`.
- **`:app` module**: depends on `:llama` and `litertlm-android`.
- **`:llama` module**: depends on `kotlinx-coroutines-core/android`. No AndroidX lifecycle needed. No third-party HTTP client libraries.

## Conventions

- Kotlin code style: `official` (set in `gradle.properties`).
- Namespace `com.pierfrancescocontino.sussurrato` for `:app`; `com.pierfrancescocontino.sussurrato.llama` for `:llama`.
- Tests: JUnit 4 for unit tests, `AndroidJUnitRunner` for instrumented tests. Both are placeholder-only.
- Native code style: matches llama.cpp conventions (`snake_case`, `LLAMA_` prefix for macros).
